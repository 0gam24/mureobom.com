[1. Claude Code 슬래시 명령어 (워크플로우 순서)
프로젝트 시작 (첫 세션)

/init — CLAUDE.md 초기 생성
/memory — CLAUDE.md / 오토메모리 편집
/mcp — MCP 서버 연결·OAuth 관리
/agents — 서브에이전트 구성 관리
/permissions — 도구 권한 allow/ask/deny 규칙 (별칭 /allowed-tools)

작업 중

/plan — 플랜 모드 진입 (/plan fix the auth bug 식으로 작업 동시 지정)
/model — 모델 전환 (opus/sonnet/haiku)
/effort — 추론 강도 (low/medium/high/xhigh/max). Opus 4.7는 xhigh가 코딩 기본값
/context — 컨텍스트 사용량 시각화
/compact — 대화 요약으로 컨텍스트 압축
/btw — 히스토리 안 늘리는 곁가지 질문
/goal — 조건 충족까지 턴을 넘어 계속 작업

병렬 실행

/agents — 서브에이전트 매니저
/tasks — 백그라운드 작업 목록 (별칭 /bashes)
/background — 세션을 백그라운드 에이전트로 분리 (별칭 /bg)
/batch — 대규모 변경을 5~30개 단위로 쪼개 워크트리별 병렬 실행 (스킬)

배포 전 검수

/diff — 변경 사항 인터랙티브 뷰어
/simplify — 최근 변경 파일 품질·중복·효율 리뷰 후 자동 수정 (스킬)
/review — PR 로컬 리뷰
/security-review — 브랜치 diff 보안 취약점(인젝션·인증·데이터 노출) 분석

세션 간 이동

/clear — 새 대화 시작, 프로젝트 메모리 유지 (별칭 /reset, /new)
/resume — ID/이름으로 대화 재개 또는 피커 (별칭 /continue)
/branch — 현재 대화 분기 (별칭 /fork)
/teleport — 웹 세션을 터미널로 가져오기 (별칭 /tp)
/remote-control — claude.ai에서 로컬 세션 원격 제어 (별칭 /rc)

문제 발생 시

/rewind — 코드·대화를 체크포인트로 되돌리기 (별칭 /checkpoint, /undo)
/doctor — 설치·설정 진단
/debug — 디버그 로깅 후 이슈 분석 (스킬)
/feedback — 세션 컨텍스트 첨부 버그 리포트 (별칭 /bug)

2. ULTRA 시리즈 (심화 추론·리뷰)

ultrathink — 명령어가 아니라 키워드. 프롬프트나 스킬 본문에 넣으면 일회성 심화 추론 발동 (당신이 이 메시지 맨 앞에 쓴 것이 이것)
/ultraplan — 클라우드 세션에서 플랜 작성 → 브라우저 검토 → 원격/터미널 실행
/ultrareview — 클라우드 샌드박스 멀티에이전트 심층 코드 리뷰

3. 번들 스킬 (Claude가 자동 호출 가능)
/batch, /simplify, /debug, /loop(주기 반복 실행, 별칭 /proactive), /claude-api, /fewer-permission-prompts — 빌트인 명령어와 달리 프롬프트 기반이라 상황에 맞으면 Claude가 알아서 호출합니다.
4. 명령어가 아닌 "개념·방법론"
이건 슬래시 명령어가 아니라 바이브 코딩 멘탈 모델로 묶이는 것들입니다.

OODA — Observe-Orient-Decide-Act 루프. 에이전트 루프 설계(관찰→해석→결정→실행→피드백)를 설명할 때 쓰는 의사결정 프레임워크지 Claude Code 기능이 아닙니다.
ARTIFACT — claude.ai의 아티팩트 기능(코드·문서 산출물 렌더링). Claude Code 슬래시 명령어가 아니라 claude.ai/Claude 앱 쪽 개념입니다.
자주 같이 묶이는 것들: spec-driven 개발, 체크포인트(=/rewind), 세션 재개(=/resume).
