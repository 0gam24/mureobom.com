# 물어봄 (mureobom.com) — 프로젝트 기획서

> 지식iN에서 "사람들이 무엇을 궁금해하는지"를 신호로 잡아,
> 그보다 정확하고 깊은 **원본 답변**을 자동 생성·발행하는 독립 Q&A 미디어.

---

## 1. 핵심 원칙 (절대 어기지 않음)

1. 네이버 데이터는 **주제·수요 발굴 신호로만** 사용한다.
2. 지식iN 질문/답변 원문은 저장·재게시하지 않는다. (약관·저작권·중복콘텐츠 리스크)
3. 본문은 **공식 1차 출처**(법령, 정부·KFTC·FSS·공단 자료) 기반 원본만 발행한다.
4. 사람은 brief.yaml 검수(약 15분)만, 그 외 파이프라인은 무인.

## 2. 사이트 정보구조 (IA) — 4 카테고리 분리

```
mureobom.com
├─ /tax        세금        (연말정산·종합소득세·부가세·양도세)
├─ /support    정부지원금  (실업급여·근로장려금·바우처·청년지원)
├─ /loan       대출·신용   (전세·주담대·신용대출·신용점수)
└─ /insurance  보험·연금   (4대보험·국민연금·실손·연금저축)
```

- 각 카테고리 = 카테고리 랜딩 + 질문형 답변 포스트 다수
- 포스트 URL: `/{cluster}/{slug}` (질문 인텐트 기반 슬러그)
- 카테고리 간 내부링크. (계산기·제휴 CTA는 수익화 모델에서 폐기 — 아래 §7 참조)

## 3. 토픽 스캐너 점수 로직

후보(시드 키워드 또는 파생 질의)마다 0~100 점수 산출:

```
score = 100 * (
      0.40 * trend_momentum      # 데이터랩 검색 트렌드 상승 모멘텀 (최근 가중)
    + 0.30 * question_volume     # kin API date정렬 최근 N일 질문량 프록시
    + 0.20 * cluster_fit         # 시드 키워드 매칭 강도(0~1)
    + 0.10 * freshness           # 가장 최근 질문의 신선도(최근일수록 1)
) - duplication_penalty          # 이미 발행/큐에 있는 주제면 감점(최대 -40)
```

- `trend_momentum`: 데이터랩 최근 구간 비율의 7일 이동평균 대비 상승률을 0~1 정규화
- `question_volume`: kin API `sort=date&display=100` 결과 중 최근 14일 내 항목 비중
- `cluster_fit`: 후보가 어느 클러스터 시드와 얼마나 강하게 매칭되는지
- `freshness`: 최신 질문 게시일 기준 지수감쇠
- `duplication_penalty`: 기발행 슬러그/큐 토픽과의 의미 중복

상위 N개를 `topic-queue.json` 으로 출력 → brief 생성 단계로 전달.

> 가중치(`config/scoring.yaml` 권장)는 1주차 PoC 후 데이터 보고 튜닝.

## 4. 파이프라인

```
GitHub Actions (cron, 매일)
  → topic_scanner.py  (데이터랩 + kin API)
  → topic-queue.json  (상위 N개, 클러스터별)
  → [brief 생성 에이전트] briefs/{cluster}/{slug}.brief.yaml 초안
  → 사람 15분 검수/승인 (status: approved)
  → [리서치 에이전트] 공식 1차 출처 수집
  → [작성 에이전트] 원본 답변 생성(Q&A + 심화 + 표/예시)
  → [GEO 에이전트] FAQ 스키마 / llms.txt
  → [검수 에이전트] 사실확인·중복검사·금융표현 컴플라이언스
  → 자동 PR → CI 통과 시 auto-merge → 발행
```

지식iN 데이터는 1번(주제·인텐트)에서만 소비되고, 본문 생성에는 흐르지 않음.

## 7. 수익화 — Google AdSense 단독

- **유일 수익원**: AdSense 광고. 계산기 딥링크·제휴 CTA는 **운영 범위 밖** (폐기).
- **승인 전제 충족**:
  - 모든 글이 공식 1차 출처 인용 (compliance O-01)
  - 최소 본문 분량 (compliance AD-02, 권장 700자+)
  - 일반정보 고지 노출 (`disclaimer: true` 기본값)
  - 도메인 자체 보유 + Cloudflare Pages 안정 운영
  - 사이트맵·robots.txt·연락처 페이지 등 표준 메타
- **슬롯 정책**:
  - Auto Ads 미사용 (가독성 보호 + 출처 박스/FAQ 위치 보호)
  - 상세 페이지에 in-article 슬롯 2개 (본문 중간 + 출처 직후)
  - 홈/카테고리 랜딩 무광고 (정보 밀도 우선)
- **차단 카테고리**: AdSense 정책 금지 (도박·성인·폭력·약물·무기) — mureobom
  4 클러스터는 모두 안전, 단 대출 클러스터의 약탈적 표현은 compliance F-01에서
  이미 차단 중.
- **표시·식별자**: `public/ads.txt`에 publisher ID 필수.
- **컴플라이언스 규칙**: AD-01~AD-05 (compliance/checklist.md §7).

## 6. 리스크 & 모니터링

- **API 한도**: kin API 앱당 일일 호출 한도(과거 기준 25,000회 수준) — 스캐너는 시드 묶음·캐싱으로 호출 절약. 1주차에 실측 필수.
- **중복콘텐츠**: 발행 전 검수 에이전트가 자체 코퍼스 + 웹 유사도 검사. 임계 초과 시 자동 차단.
- **금융 표현 컴플라이언스**: KFTC 표시·FSS 광고심의 관점 체크리스트를 검수 단계에 내장.
- **품질 하한**: 공식 출처 인용 0건이면 발행 차단(원본성 가드).
