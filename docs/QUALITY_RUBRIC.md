# 품질 게이트 채점 루브릭 (100점, 게이트 85점)

quality-gate 에이전트가 발행 전 모든 글을 채점한다. 작성자(writer)와 분리된
컨텍스트에서 레드팀 관점으로 채점하며, 글을 직접 고치지 않고 판정만 한다.

## 1. 사실 정확성·출처 검증 — 30점
- [10] 글의 핵심 수치·기간·자격 요건이 1차 출처(정부24, 복지로, 국세청, 보조금24, 법제처)와 일치
- [8] 모든 출처 URL이 실재하고(WebFetch로 표본 2개 이상 실확인) 해당 내용을 실제로 담고 있음
- [6] 시효 표기: 기준 연도/신청 기간이 본문에 명시되고 현재 유효
- [6] 추측·과장 표현 부재 ("~로 알려져 있다", "무조건", "100%" 류 감점)
- 치명 결격(즉시 0점 처리): 존재하지 않는 제도·지원금 서술, 위조 출처

## 2. 고유 가치 — 25점
- [10] 고유 데이터 요소 1개 이상: 직접 구성한 비교표, 지역별 수치, 계산 예시, 시나리오별 분기
- [8] 고유 해석 레이어: 출처 대조에서 나온 판단·주의점·실수 포인트 (출처 요약 짜깁기만 있으면 0)
- [7] SERP 상위 문서들이 다루지 않는 각도 1개 이상 (brief의 SERP 분석 대비)

## 3. 검색의도 충족·구조 — 20점
- [8] 리드 답변: 첫 화면에서 질문에 대한 핵심 답 3줄 이내 제시 (Q&A 포맷 필수)
- [6] 절차·자격·기간·금액이 표 또는 단계로 스캔 가능
- [6] FAQ 3개 이상 + FAQPage 스키마, 제목이 실제 검색 질문형

## 4. 정책 준수 — 15점
- [6] YMYL 프레임: 조언·권유 없음, 정보·절차만. "신청하세요" 대신 "신청할 수 있다"
- [5] 애드센스 정책: 클릭 유도 문구 없음, 과장 제목(어그로) 없음, 금지 주제 비접촉
- [4] 출처 블록 + "정보 기준일" 명시, 운영자/문의 경로 접근 가능

## 5. 중복·내부링크 — 10점
- [5] 자사 기발행 글과 주제 중복 없음 (automation/briefs/_published_slugs.txt 슬러그 대조 + src/content/answers/*.md 프론트매터 grep), 본문 표현 중복률 낮음
- [5] 관련 내부링크 2개 이상 (실재하는 자사 URL만)

## 판정 규칙
- 85점 이상: PASS → 발행 진행
- 70~84점: REVISE → 감점 사유를 writer에 반환, 리비전 1회만 허용. 재채점 후 85 미만이면 KILL
- 70점 미만 또는 치명 결격: KILL → 발행 폐기, 사유를 장부에 기록

## 출력 포맷 (quality-gate 에이전트 필수 준수)
```json
{
  "verdict": "PASS | REVISE | KILL",
  "score": 0,
  "breakdown": {"accuracy": 0, "unique_value": 0, "intent": 0, "policy": 0, "dedup_links": 0},
  "fatal": false,
  "issues": ["감점 사유를 구체적 위치와 함께"],
  "checked_sources": ["실확인한 출처 URL"]
}
```
